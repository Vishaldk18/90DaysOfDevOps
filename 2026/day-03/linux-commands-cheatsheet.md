Linux Commands Cheat Sheet — Process Management,
File Management & Networking
A concise but detailed reference of essential commands with their most useful options.
Generated by M365 Copilot
1) Process Management
ps — process status
Display information about active processes. BSD (aux) and System V (-ef) styles are common.
# Common usages
ps # processes in current shell/TTY
ps -ef # all processes, full format (System V)
ps aux # all processes (BSD style)
ps -u <user> # processes for a user
ps -p <PID1,PID2> # specific PIDs
ps -o pid,ppid,%cpu,%mem,cmd # custom columns
ps -ef --forest # tree view
top — real-time process monitor
Interactive process viewer with CPU/memory summaries.
# Flags
top -u <user> # filter by user
top -d 2 # refresh every 2 seconds
top -b -n 1 # batch mode, one iteration (scripting)
# Keys (inside top)
P/M/T # sort by CPU/memory/time
1 # per-CPU breakdown
k # kill by PID
r # renice
q # quit
htop — enhanced interactive process viewer
Colorful, scrollable, mouse-friendly alternative to top (may need installation).
# Start and basics
htop # launch
htop -u <user> # filter by user
htop --sort-key=MEMORY # choose initial sort key
# Keys
F3 search F5 tree F6 sort F7/F8 renice F9 kill F10 quit
pgrep / pkill — find or signal processes by name
pgrep searches PIDs by name/attributes; pkill sends signals to matching processes.
pgrep -fl <pattern> # list PIDs with command lines
pgrep -u <user> <pattern> # match by user
pkill -f <pattern> # kill by full command match
pkill -TERM <name> # send specific signal
pkill -9 <name> # force kill (SIGKILL)
kill / killall — send signals to processes
Terminate or control processes using POSIX signals.
kill -TERM <PID> # graceful stop (default)
kill -KILL <PID> # force kill
kill -STOP <PID> # pause a process
kill -CONT <PID> # resume a process
killall <name> # signal by process name
nice / renice — set CPU scheduling priority
Lower priority (positive nice) yields CPU to others; negative nice requires privileges.
nice -n 10 command # start with lower priority
sudo renice -n -5 -p <PID> # increase priority (requires root)
nohup / disown / & — run processes detached
Keep processes running after logout or run in background.
command & # background job
nohup command > out.log 2>&1 & # immune to hangups
# In bash job control
jobs; fg %1; bg %1; disown %1
strace / ltrace — trace system or library calls
Debug what a process is doing at the syscall or libc layer.
strace -p <PID> # attach to a running process
strace -o trace.txt command # write trace to file
ltrace -p <PID> # trace library calls
systemctl / service — manage systemd services
Control services and review status/logs on systemd-based distros.
systemctl status <svc>
systemctl start|stop|restart <svc>
systemctl enable|disable <svc>
journalctl -u <svc> --since "1 hour ago"
2) File System & File Management
pwd / cd / ls — navigate and list
Move around the filesystem and list directory contents.
pwd
cd /path; cd ..; cd ~
ls -l # long format
ls -a # include hidden
ls -lh # human sizes
ls -ltr # newest last
touch / mkdir / rmdir — create and remove
Create files and directories.
touch file.txt
mkdir mydir
mkdir -p parent/child # nested
rmdir emptydir # remove empty dir
cp / mv / rm — copy, move, delete
Copy/move files and directories; remove carefully.
cp file1 file2
cp -r dir1 dir2 # recursive copy
mv file1 /path/ # move or rename
rm file.txt
rm -r folder # recursive
rm -rf folder # force + recursive (danger!)
cat / tac / head / tail / less — view files
Quickly inspect files; less supports paging and search.
cat file.txt
head -50 file.txt
tail -f app.log # follow log
less largefile # /search n/N next/prev
grep / egrep / ripgrep — search within files
Find text patterns with regular expressions.
grep -R "error" /var/log
grep -i "failed" *.log
grep -E "foo|bar" file.txt # extended regex
rg -n "pattern" . # ripgrep if installed
find / locate — find files by attributes
Locate files by name, size, time, type, and run actions.
find /path -name "*.log"
find . -type f -size +10M -mtime -2 -print
find . -type f -exec grep -H "TODO" {} +
updatedb; locate foo.txt
chmod / chown / chgrp — permissions & ownership
Change mode bits and file ownership.
chmod 755 script.sh
chmod u+x script.sh
chown user:group file
chgrp developers file
ln — hard and symbolic links
Create references to files; symbolic links can cross filesystems.
ln file linkname # hard link
ln -s /path/target link # symbolic link
du / df / stat — sizes and filesystem info
Check usage and stats of files and filesystems.
df -h # disk free (human)
du -sh * # sizes per item
stat file.txt # inode, mode, timestamps
tar / gzip / bzip2 / xz — archive & compress
Create and extract archives; choose compressors by tradeoffs.
tar -cvf archive.tar dir/
tar -xvf archive.tar
# With compression
tar -czvf archive.tar.gz dir/ # gzip (fast)
tar -cjvf archive.tar.bz2 dir/ # bzip2 (smaller)
tar -cJvf archive.tar.xz dir/ # xz (smallest)
# Extract
tar -xzvf archive.tar.gz
zip / unzip / 7z — alternate archiving tools
Cross-platform zip archives; 7z often gives best ratios.
zip -r files.zip dir/
unzip files.zip
7z a files.7z dir/
7z x files.7z
3) Networking & Diagnostics
ip / ifconfig — interfaces, addresses, routes
`ip` is the modern suite; `ifconfig` is legacy but still encountered.
ip a # addresses
ip link # links
ip route # routing table
ip addr add 192.168.1.5/24 dev eth0
ip link set eth0 up|down
ifconfig # legacy
ping / traceroute / tracepath — connectivity
Check reachability and view path to a host.
ping -c 5 8.8.8.8
traceroute example.com
tracepath example.com
curl / wget — HTTP/HTTPS testing and downloads
Inspect endpoints, headers and download files.
curl https://example.com
curl -I https://example.com # headers only
curl -sS -o out.json https://api.example.com/data
wget https://example.com/file.iso
dig / nslookup / host — DNS queries
Resolve names, view records and authoritative answers.
dig example.com +short
dig @8.8.8.8 example.com ANY
nslookup example.com
host -t A example.com
ss / netstat — sockets and ports
List listening sockets, connections and owning processes.
ss -tulpn # TCP/UDP listening with PIDs
ss -tnl # TCP listen only
ss -s # summary stats
netstat -tulpn # legacy equivalent
nc (netcat) / telnet — test ports and transfer
Swiss-army knife for TCP/UDP; simple listeners and port checks.
# Check port reachability
nc -zv host 443
# Simple TCP server and client
nc -l 1234 > received.txt
# On another host
nc server 1234 < file.txt
# Telnet quick test
telnet host 80
tcpdump — packet capture
Capture and filter packets for troubleshooting (requires privileges).
tcpdump -i eth0
tcpdump -i any port 443
tcpdump -w capture.pcap # save for Wireshark
iperf3 / speedtest-cli — bandwidth tests
Measure throughput locally (iperf3) or to internet test servers (speedtest).
iperf3 -s # server
iperf3 -c <server_ip> # client
speedtest-cli # internet speed test
Firewall tools — ufw / firewall-cmd / iptables-nft
Allow/deny ports and inspect rules depending on distro.
ufw status
ufw allow 80/tcp
firewall-cmd --state
firewall-cmd --list-all
sudo iptables -L -n -v
hostname / nmcli — host & NetworkManager
Quick host identification and NetworkManager control.
hostname
hostname -I # IPs assigned
nmcli device status
nmcli connection show
Note: Some commands may require root privileges (use sudo). Options can vary slightly by distribution.
